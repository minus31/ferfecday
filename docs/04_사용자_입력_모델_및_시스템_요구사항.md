# 4장. 사용자 입력 모델 및 시스템 요구사항
앱 개발 시 데이터 입력 UI/UX 설계와 백엔드 API 연동을 위한 규격 요구사항입니다.
## 4.1 필수 사용자 입력 항목 (User Inputs)
- 아이의 생물학적 성별 (Gender)
- 선택 옵션: 남성(Male) / 여성(Female)
- 시스템 필요성: 대운(大運)의 순행(Forward) 및 역행(Backward) 방향을 결정짓는 필수 기준값입니다. (연간 천간의 음양과 성별 조합 활용)
- 출생 예정 지역 (Birth Location)
- 입력 방식: 국내 시/도/구 군 선택 또는 해외 주요 도시 검색 UI 제공.
- 시스템 필요성: 서울 표준시(동경 135도) 기준 대비 실제 출생 지역의 경도(Longitude) 편차를 계산하여, 실제 태어날 시각(자시~해시)의 경계선을 정확히 보정하는 데 활용합니다. (예: 부산과 서울의 실제 태양시 차이 보정)
- 출생 예정 기간 (Target Delivery Period)
- 입력 방식: 달력(Calendar) 인터페이스에서 연속된 3일 ~ 최대 5일의 예정 기간 선택.
- 시스템 필요성: 연산 대상이 되는 타임스탬프 범위를 설정하고 사주 생성을 위한 스케줄러 인풋값으로 활용합니다.
## 4.2 인프라 및 엔진 요구사항
- API Latency: 3~5일간의 총 조합(최대 60개 사주 세트)에 대해 천간/지지의 가중치, 합충 변화, 용신 추출, 가감점 모델 연산 및 정렬이 1.5초 이내에 완료되어야 합니다.
- 만세력 엔진 규격: 양력/음력 변환, 24절기 절입 시각(분 단위) 정밀 계산 모듈이 내장되어야 합니다.
